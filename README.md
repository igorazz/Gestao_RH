##Conceitos de POO em Java.

No desenvolvimento desse pequeno sistema de gestão de departamentos, apliquei conceitos fundamentais que fazem toda a diferença na escalabilidade de um projeto:

✅ Encapsulamento: Protegendo dados com atributos private e controlando o acesso via Getters e Setters. 

✅ Associação de Classes: Entendendo como um objeto Department pode "ter um" Employee, criando relacionamentos lógicos entre entidades.

✅ Separação de Responsabilidades: Criando classes específicas para modelos de dados e outras para lógica de negócio, como o DepartmentFinder. 

✅ Construtores: Garantindo a integridade dos objetos desde o momento da sua criação.
